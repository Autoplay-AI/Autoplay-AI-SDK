# Governance

## Project model

`autoplay-sdk` is currently maintained by the Autoplay AI maintainers.

Maintainers are responsible for:

- Reviewing and merging pull requests
- Managing releases and changelog quality
- Triaging issues, support requests, and security reports
- Enforcing the Code of Conduct and security policies

## Decision making

- Day-to-day decisions are made by maintainers
- For high-impact or breaking changes, maintainers seek explicit review and
  consensus in pull requests before merge
- Changes are expected to include tests and changelog entries where applicable

## Release process

- Releases are versioned in `pyproject.toml`
- Notable changes are documented in `CHANGELOG.md`
- PyPI publishing is handled by the pinned and reviewed `publish.yml` workflow

## Becoming a maintainer

New maintainers are invited by existing maintainers based on sustained,
high-quality contributions and trusted stewardship in code review and support.
