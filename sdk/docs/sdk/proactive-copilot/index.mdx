✅ **A proactive chatbot is different.** It watches, learns, and acts — surfacing the right next step before users have to ask.

## The building blocks

Follow the steps in order — each page stands alone so you can ship incrementally.

1. **[Connect real-time actions](./step-1-connect-real-time-actions)** — Stream structured `ActionsPayload` events into your stack.
2. **[Add actions to LLM context](./step-2-add-actions-to-llm-context)** — Combine live actions, chat history, and optional KB into one inference-ready context.
3. **[Define proactive triggers](./step-3-define-proactive-triggers)** — When to interrupt, then **deliver** as proactive **chat / quick reply** (SDK) or **on-screen visual guidance** (BYO tours); registry, FSM gating, and routing **chat vs tour vs silence** in your handler.
4. **[User memory and workflow ontology](./step-4-enhance-with-user-memory)** — Define workflows so Atlas and memory expose **completion rates** and mastery; then filter with cross-session **user memory**.

You can build and ship real-time event ingestion today. **Workflow ontology** (Chrome extension; docs with Autoplay via Slack) is part of [Step 4](./step-4-enhance-with-user-memory); expect an **iterative loop** on **Autoplay Atlas** labelling quality. The in-SDK [knowledge base query](../knowledge-base), full user memory rollout, and visual tours are still rolling out.
